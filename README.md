How do I interpret user needs and implement them into a program? How does creating "user stories" help with this?

  Creating User Stories helps interpret user needs by giving us real-world examples of the user needs and viewing from their perspective. Reading some or hearing someone else's perspective helps us learn and see things from another's view. We often see things and build software to view them and not from a user's standpoint. A user needs to come in and use the software without knowing anything about how it was built. I feel like this is the most significant benefit of the "user stories" from a developer standpoint.
